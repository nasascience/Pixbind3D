Popups and Dialogs

An example of a tab control:
  • Input
  • Information
  • Error
  • Success
  • Warning
  • Confirmation
  • Helper
  • Dialogs should be no bigger than 65% of the screen size
  • Dialogs should not contain expandable content that might, when expanded, force it to be bigger than 65% of the screen
  • When displayed, the screen behind the dialog should be greyed out, and should not respond to any mouse clicks.
  • Prefer a new page to a popup for requesting information that will be stored to the database.
  • When a popup does immediately commit a change to the database, ensure the button is titled ‘Save’, not ‘OK’.
