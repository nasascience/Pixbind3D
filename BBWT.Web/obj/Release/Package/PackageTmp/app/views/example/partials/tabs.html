Tabs

Use tabs to alternate between attributes belonging to, or strongly associated with, an instance of a particular object. Do not use them to navigate to the management of objects that are only associated with the type of your object.

E.g. Imagine that your “reports” have a set of “parameters” associated with each report, i.e. a parameter “Ordering Company” is a filter for the “Banana Orders” report. The reports also have a set of “report users” that are allowed to view any report, i.e. “John Smith” is allowed to see any of the reports. Note that “report users” are associated with “reports” in general, not one particular report. In this case, your Edit Report page might have a tab for parameters, but it should not have a tab for the report users. Instead, there should be a “Manage Report Users” page.

An example of a tab control:
  • General
  • Misc
  • Advanced
Tab with general settings.
Tab with misc settings.
Tab with advanced settings.
The HTML looks like this: 
<div kendo-tab-strip k-animation="false">
    <ul>
        <li class="k-state-default k-state-active" k-state-disabled>General</li>
        <li>Misc</li>
        <li>Advanced</li>
    </ul>
    <div>
        <div class="panel-body">
            Tab with general settings.
        </div>
    </div>
    <div>
        <div class="panel-body">
            Tab with misc settings.
        </div>
    </div>
    <div>
        <div class="panel-body">
            Tab with advanced settings.
        </div>
    </div>
</div>
  • kendo-tab-strip is used to create tabs
  • The tabs are <li> items in a <ul>.
  • Each tab pane is a <div>
  • Javascript is not needed.
